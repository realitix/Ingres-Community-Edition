Connect to an Ingres Instance

Before it can retrieve data, the demonstration application must first establish a connection with the server that manages the demodb database.

The Ingres DBMS Server can exist locally (on the same machine as the application) or remotely (on a different machine than the application). Initially, it is assumed that the application and data source are on the same machine.

To change the data source

  1. Check the Defined source button.

    The fields are enabled so you can modify the connection details.

  2. Provide information in the following fields:
    • Host Name

      Specifies the name or address of the default target host. The default is (local).

    • Instance name

      Specifies the unique Ingres identifier, a component of an Ingres instance name. The default is II.

    • Database name

      Target database name. The default is demodb.

      Note: This field cannot be updated for this application.

    • User ID

      The name of the authorized user connecting to the Ingres DBMS Server.

      Note: This information is required only when connecting to a remote Ingres database.

    • Password

      The password associated with the specified user ID for connecting to the Ingres DBMS Server.

      Note: This information is required only when connecting to a remote Ingres database.

  3. (Optional) Click Make defined source default.

    The newly-defined data source becomes the default one to connect to.

  4. Click Change.

    You are connected to the target DBMS Server.
