Building ICESAX.DLL

Introduction:

The decision has been made to rename the xerces dynamic link library so as to avoid possible conflicts with other SAX parsers in the field. It will also discourage people dropping other versions of the xerces library on top of the one supplied for ICE, we hope.

Building the ICE SAX DLL:

Getting the Apache Xerces C SAX Parser

Visit apache web site and download the source code for the Xerces C++ SAX Parser. ICETranslate has been designed to work with version 1.4.0 of the parser.

Special Build Instructions for the Xerces Parser

To build the dll on in MS VC on NT you open the project xerces-all.
  • Go to Project Settings
  • Ensure that 'All Configurations' is selected in the drop down on the Left Hand Pane
  • In the Right Hand Pane select Tab Link
  • In the edit box 'Output File' specify target name to be 'icesax.dll'

Debugging Gotcha:

To avoid debugging woes click the tab C++, select Code Generation from the combo box. Then ensure that the 'Multithreaded DLL' has been selected for 'All Configuration' and 'Multithreaded debug DLL' for 'Debug' in Project Settings.

Note that there are problems using a debug enabled ICETranslate with a non-debug library. If you need to debug the executable, you will need to get hold of the XERCES source code and build it for debug.
