When documents are stored in the database, extracting them on each client request can greatly increase response time to the browser. This time lag increases as the size of the document to be extracted increases.

To improve performance, Web Deployment Option utilizes a caching system to reduce the number of extractions from the data source. Three different levels of cache can be set for any individual document by that document's owner. These include:

Pre-cache:
When the ICE Server is started up, the document is cached
until the server is shut down.
Permanent cache:
The first time a document is requested by a client, the document
becomes "permanently" cached until the server is shut down.
Session cache:
A document is only cached during a client session.
As soon as the session is terminated, the document is erased
from the cache if it has not been requested by another client.
