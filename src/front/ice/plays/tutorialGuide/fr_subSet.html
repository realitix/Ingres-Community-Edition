Displaying a Subset of Rows

This document displays those plays required, as specified by the variable e_type. This is used in the where clause of the select statement: 
          SQL=`select * from plays where type = ':e_type'`

We also include a link back to the type selection page: 

<A HREF="/ice-bin/oiice.dll/playgroup/plays[play_typeList.html]">Select a new play type</A>

In every other respect, this document is identical to the first example we saw. We leave to the reader the exercise of extending this example to wrap around to the beginning when the end of the result set is reached.

File: play_subSet.html

<html>
<head>
<title>Shakespeare's Plays by Type</title>
</head>
<body>
<h1>Shakespeare's Plays by Type</h1>

<FORM ACTION="/ice-bin/oiice.dll/playgroup/plays[play_subSet.html]"
	METHOD="POST">
<!-- #ICE 
          DATABASE = `icetutor` 
          SQL=`select * from plays where type = ':e_type'` 
          TRANSACTION=`Complete` 
          CURSOR=`Works` 
          ROWS=`5` 
          TYPE=`TABLE`
          -->
<!-- #ICE DECLARE=`session.e_playTxn=Complete` -->
<!-- #ICE DECLARE=`session.e_playTxnCommitted=FALSE` -->
<p>
<A HREF="/ice-bin/oiice.dll/playgroup/plays[play_typeList.html]">Select a new play type</A>
<p>
<INPUT TYPE="submit" NAME="More" VALUE="More" ALT="Show More Plays of this type">
</FORM>
<!-- #ICE REPEAT INCLUDE=`plays[play_SessionControl_h.html]` -->
</body>
</html>

In this example we have seen how we can communicate values between two (or more) Web Deployment Option HTML documents, presenting a simple refinement of search criteria to the user, based on data in the database.
