Create the Welcome Page

The first page which the user encounters is the welcome page. This page lies outside the control of the ICE server and is a normal HTML page.

The welcome page is needed to provide access to the pages that do come under the control of the ICE server. It contains some welcome text and a link to the first page, which is a login page and, as a result, must have public access.

Here is an example:

File: play_welcome.html

<html>
<head>
<title> Shakespeare Live </title>
</head>
<body>
<center>
<h1> Shakespeare Live </h1>
</center>
<center>
Welcome to the <b>globe</b>
site for Shakespeare Studies.
<p>
<a href=/ice-bin/oiice.dll/plays[play_login.html]><img src="OldGlobe.gif" alt="Enter the Globe Here"</a>
</center>
</body>
</html>
The line of interest here is the anchor:

<a href=/ice-bin/oiice.dll/plays[play_login.html]>Enter the Globe Here</a>

This tells the ICE server to access the play_login page which resides in the plays location. The brackets in the syntax show that play_login.html is a page in the plays location.

There is no support for Session Groups at this level because we have not yet logged in and therefore have not yet been asigned a session id.

The next page is the login page.
