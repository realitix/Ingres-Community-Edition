Page to Automatically Create a New Account

Anyone accessing non public pages in a part of your web site controlled by Web Deployment Option must have an account. Since this could mean creating accounts for a large number of users, we need some way for the users to allocate themselves an account. The account should have the minimum privileges required, by associating a role in the administration tool.

Here is an example of a self declaration page:

File: play_autoUser.html

<HTML>
<HEAD>
<TABLE BORDER=0 WIDTH="100%">
	<TD><IMG SRC="/iceimages/ingreslogo.gif"></TD>
	<TD ALIGN=CENTER><H1>Automatic Declaration</H1></TD>
</TABLE>
<HR>
</HEAD>

<BODY BGPROPERTIES=FIXED BGCOLOR=WHITE TEXT=#000000 LINK=#ff6600 VLINK=#330099>
 	<FORM ACTION="/ice-bin/oiice.dll/plays[play_home.html]" METHOD="POST">
		<INPUT TYPE=hidden NAME="ii_action" value="declare">
		<INPUT TYPE=hidden NAME="ii_profile" value="play_tutor_profile">
		<CENTER>
		<TABLE BORDER=0 ALIGN=CENTER VALIGN=CENTER>
			<TR>
				<TD>Please enter your:</TD>
			</TR>
			<TR>
				<TD>Name: </TD>
				<TD><INPUT SIZE=32 NAME="ii_userid"></TD>
			</TR>
			<TR>
				<TD>Password: </TD>
				<TD><INPUT SIZE=32 TYPE=PASSWORD NAME="ii_password"></TD>
			</TR>
		</TABLE>		
		<INPUT TYPE="IMAGE" BORDER=0 NAME="connect" SRC="/iceimages/logout.gif" ALT="Connect">
		</CENTER>
	</FORM>
</BODY>
</HTML>

In this example, the lines of interest are


<FORM ACTION="/ice-bin/oiice.dll/plays[play_home.html]" METHOD="POST">
<INPUT TYPE=hidden NAME="ii_action" value="declare">
<INPUT TYPE=hidden NAME="ii_profile" value="play_tutor_profile">

The action for the form is the document play_home.html in the location plays. It is passed, along with the new user name and password, the action declare. This causes a new web user to be created with the profile play_tutor_profile. Note that the profile play_tutor_profile must have been previously defined in the administration tool.
