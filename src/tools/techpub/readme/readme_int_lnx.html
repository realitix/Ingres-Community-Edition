1.0 Welcome

This readme contains information specific to Ingres running under 32-bit Linux with NPTL support.

2.0 Operating System Support

The following operating systems are supported (as of the publishing of this document):

●       Red Hat Enterprise Linux 6

●       Red Hat Enterprise Linux 5

●       Red Hat Enterprise Linux 4

●       SUSE Linux Enterprise Server 11.x

●       SUSE Linux Enterprise Server 10.x

●       SUSE Linux Enterprise Server 9.x

Kernel and glibc requirements:

●       32-bit Linux Kernel 2.4.21-20EL and up with glibc 2.3-95 C-language libraries and up with NPTL support.

●       32-bit Linux Kernel 2.6 with glibc 2.3 to 2.7 C-language libraries with NPTL support.

The Ingres product support life cycle policy determines the duration of support for Ingres on these operating systems unless the operating system's manufacturer drops support on a prior date. For the latest information about supported operating systems, visit http://www.actian.com/support.

Note: When installing on Red Hat Enterprise Linux 4.0 or later, the following packages must be installed for Ingres to function correctly:

compat-gcc-32
compat-gcc-32-c++
compat-libstdc++-296
compat-libstdc++-33

When installing on SUSE Linux Enterprise Server 9.x systems, the following package must be installed:

libstdc++-3.3.3

When installing on SUSE Linux Enterprise Server 10.x systems, the following package must be installed:

compat-libstdc++-5.0.7

2.1 Preprocessor and Compiler Support

●       GCC 3.3.3 to 4.3.3 (C, C++, Fortran, and Ada)

4.0 Installation Considerations

4.1 RPM Support

RPM 4.1.1 or later

4.2 Installing Ingres Using RPM

To install Ingres using the new graphical installation wizard, run the script ingres_install found in the directory created when the saveset is extracted.

Note: The installation wizard is not supported on 64-bit Linux. To install Ingres, use ingres_express_install.sh or rpm directly. For more information, see the Quick Start Guide.

The installation process for installing Ingres using RPM has changed in Ingres 10.1. When installing Ingres using the RPM command, the setup portion of the installation process must be completed by running the service script. The service script configures and starts the instance.  For more information, see the Installation Guide.

4.3 How to Upgrade Using RPM

Because of a number of problems observed during upgrade of Ingres RPMs, please consult the instructions in the Installation Guide before upgrading. 

●       BUG 115822Upgrading an RPM installation that has been installed in a location other than the default can result in all installed files (except data files and generated configuration files) being removed. This is caused by a bug in RPM version 4.1.1.

4.4 Ingres Cluster Solution

The Ingres Cluster Solution does not support active-active clustering.

To configure Ingres for the High Availability for Red Hat Cluster Suite, see the Installation Guide.

4.5 Post Installation

The only supported method for backing out patches installed against Ingres 10.1 is to reinstall the RPM from the base release. Therefore, make sure you keep the RPM used for your most recent install so that a patch can be backed out, if necessary.

4.6 Documentation

Ingres documentation can be installed on Linux systems by installing the documentation RPM file that can be downloaded separately.

4.7 Visual Tools

The Visual DBA suite is not included as part of the Ingres 10.1 for 32-bit Linux. However, these tools are included in Ingres 10.0 for Windows client and can be installed on a Windows PC to administer Ingres 10.0 for 32-bit Linux remotely.

The Visual DBA suite is also available separately as the Community Ingres DBA Tools package and is available for download from esd.actian.com.

The Ingres Database Workbench (IDW) is available for Linux and lets you browse your database objects such as tables, views, procedures, and rules. You can also issue SQL statements using the SQL scrapbook editor, edit the contents of your tables, and edit your procedures and rules. IDW is available for download from esd.actian.com.

4.8 Protocols Supported

TCP/IP Native

4.9 Raw Transaction Log Files and Raw Database Locations Not Supported on 32-bit Linux

Research has shown that raw transaction log files and raw database locations on 32-bit Linux do not provide performance benefits. To simplify administration, this feature is not supported on Ingres 10.0 for 32-bit Linux.

4.10 Oracle Tuxedo Support

Oracle Tuxedo 10.0 is supported with this version of Ingres. Please visit http://www.oracle.com/products/middleware/bea.html for more information.

5.0 General Considerations

5.1 Ada Preprocessor

The Ada preprocessor has not been enhanced to support new DBMS functionality in Ingres 10.x. There are no plans to enhance the Ada preprocessor to support either new language features or new DBMS functionality in Ingres 10.1 or future releases.
